# Foundations: Principles

## Goals
- Build components that stay consistent with Figma and token contracts.
- Prevent style drift from hardcoded values.
- Keep component APIs predictable and composable.

## Non-Negotiables
- No hardcoded visual values unless a documented exception exists.
- All visual styling must map to semantic tokens.
- Accessibility semantics and keyboard support are required.
- Story/Explorer behavior must reflect shipped API behavior.
